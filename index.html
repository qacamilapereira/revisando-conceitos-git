Conceitos básicos

Iniciando o Git no Projeto

git init

Adicionando num estágio para commit

git add .

Para adicionar apenas um arquivo

git add NomeDoArquivo.extensão
git status

Primeiro commit

git commit -m 'Primeiro commit'

Visualizando o histórico de commits

git log
Para sair do modo de visualização do git log no VS Code, pressione a tecla q. Isso funciona porque o git log utiliza um programa como o less por padrão, que permite rolar o conteúdo, e o comando q encerra essa visualização e o programa do pager.

Para ver as diferenças

git diff

Como descartar as alterações? E não quer "commitar"

No caso de modificar errado.
git restore NomeDoArquivo.extensão
Ele tira as modificações feitas e volta para última versão.

Criando e mesclando branches

Cada pessoa que está trabalhando no projeto cria as suas ramificações.
git branch (para mostrar em qual branch está)
Para criar uma nova branch>
git checkout (saindo da branch que está e indo para uma nova)
git checkout -b nome-da-branch (cria uma nova branch a partir dessa que está)
git checkout main (para voltar para a master)
git merge nome-da-branch (para enviar as alterações de uma branch para a branch principal)

Repositórios remotos (ou cloud/nuvem)

Publicar branch no Github, por exemplo.
Outros exemplo: GitLab, ...
Crie um novo repositório no seu GitHub para o seu projeto.
